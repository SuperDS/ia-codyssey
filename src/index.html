기존 이미지를 기반으로
나만의 Docker 웹 랜딩페이지 제작

본 페이지는 NGINX 공식 이미지를 기반으로 제작한 커스텀 이미지 실습 결과물입니다. 정적 콘텐츠 교체 방식으로 index.html을 수정하여, 코디세이 1차 과제의 목적과 실습 내용을 한눈에 볼 수 있는 랜딩페이지 형태로 구성하였습니다.

주요 기능 보기 과제 요약 보기

이번 과제에서 구현한 내용

기존 NGINX 웹 서버 이미지를 활용하고, 기본 정적 페이지를 직접 제작한 index.html로 교체하여 커스텀 이미지 빌드 및 실행에 성공하였습니다.

기존 베이스 이미지 활용

NGINX 공식 이미지를 기반으로 선택하여, 기존 웹 서버 환경을 그대로 활용하면서 정적 콘텐츠만 교체하는 방식으로 커스터마이징을 진행했습니다.

정적 콘텐츠 교체

기본 welcome 페이지 대신 과제 목적에 맞는 랜딩페이지형 index.html을 작성하고 웹 루트 경로에 복사하여 새로운 화면이 보이도록 구성했습니다.

커스텀 이미지 빌드

Dockerfile을 작성한 뒤 빌드 명령을 통해 새로운 이미지를 생성하고, 해당 이미지가 정상적으로 만들어지는 것을 확인했습니다.

컨테이너 실행 확인

생성한 이미지를 기반으로 컨테이너를 실행하고, 브라우저에서 랜딩페이지가 정상적으로 노출되는지 확인했습니다.

과제 요약

이번 실습은 단순히 이미지를 실행하는 것을 넘어, 기존 이미지를 기반으로 직접 내용을 바꾸고 그 결과를 서비스 형태로 확인했다는 점에 의미가 있습니다.

  • 기존 베이스 이미지: nginx:latest
  • 커스텀 방식: 정적 HTML 콘텐츠 교체
  • 핵심 파일: Dockerfile, index.html
  • 빌드 결과: 커스텀 이미지 생성 성공
  • 실행 결과: 컨테이너 실행 및 웹 페이지 노출 성공
  • 학습 포인트: 이미지 / 컨테이너 / 웹 서버 동작 이해
